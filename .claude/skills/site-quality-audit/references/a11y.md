# Accessibility invariants

These properties already hold in the codebase. The audit's job is to verify
they still hold — especially after new sections, animations, or interactive
elements are added.

## Structure & navigation

- **Skip link** — `app.component.html` starts with `.skip-nav` →
  `#main-content`. Any layout rewrite must keep it first-focusable.
- **Heading order** — exactly one `h1` (hero name), `h2` per section title,
  `h3` for cards/items. New sections follow the same ladder.
- **Landmarks** — `<header role="banner">`, one `<main>`, sections carry
  `aria-label` (from `sectionLabel()` in `dynamic-single-container`).

## Interactive patterns in use (preserve them)

- Language toggle: `aria-pressed` reflects the active language, per-button
  `aria-label` (`header.component.html`).
- Theme toggle: `aria-label` describes the *action* ("Switch to dark mode"),
  not the state.
- Hamburger: `aria-expanded` + `aria-controls="mobile-nav"`; mobile nav uses
  `inert` + `aria-hidden` when closed; Escape closes and returns focus to the
  toggle (`header.component.ts` `closeMenuOnEscape`).
- External links: `target="_blank"` always paired with
  `rel="noopener noreferrer"` and an `aria-label` naming the destination.
- Decorations: three.js canvas, cursor glow, scroll progress, floating shapes
  are all `aria-hidden="true"` — anything purely visual added later must be too.

## Motion (WCAG 2.2.2 / 2.3.3)

`prefers-reduced-motion: reduce` must keep disabling, in every code path:

- hero typing loop → static first phrase (`hero.component.ts`)
- GSAP name/parallax/cursor-glow animations (`hero.component.ts`, `app.component.ts`)
- the entire three.js scene (`scene-background.component.ts` skips init)
- scroll-reveal → sections shown immediately (`scroll-animation.service.ts`)
- magnetic hover directive → no-op (`magnetic-hover.directive.ts`)

Any **new** animation must ship with its reduced-motion branch in the same PR.

## Color & content

- Accent-as-text must use `--color-accent-text` (AA-verified per the comments
  in `styles.scss`), never raw `--color-accent`. Fills under white text use
  `--color-accent-solid`.
- Every `<img>` gets a meaningful `alt` sourced from the descriptor (company,
  institution, project name). Empty `alt` only for pure decoration.
- Both themes must be checked — dark theme has its own token block in
  `styles.scss` (`[data-theme="dark"]`).

## Verification procedure

1. Keyboard-only pass: Tab through header → hero CTAs → socials → project
   links → contacts; focus must always be visible; Escape closes mobile menu.
2. Toggle `prefers-reduced-motion` in devtools rendering panel → no typing
   loop, no canvas animation, sections all visible.
3. Quick automated pass on the built site:
   `npx http-server dist/naw/browser -p 8080` then run Lighthouse a11y
   category or axe devtools; triage anything below 100.
